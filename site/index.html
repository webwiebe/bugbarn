Error tracking SaaS charges by the event. One bad deploy and your bill spikes. BugBarn runs on your own server, stores everything in SQLite, and costs nothing beyond your infrastructure.

# Features

Issue grouping

Events with the same root cause group into one issue. Fingerprinted by exception type, message, and stack trace. No noise.

Alerts

Webhook alerts to Slack, Discord, or any HTTP endpoint. Three conditions: new issue, regression, event count threshold.

Weekly digest

One email per week covering all your projects. Stats, top issues, links. Configurable day and hour.

Live log stream

Ingest structured logs alongside errors. Stream them live in the UI over SSE.

Multi-project

One BugBarn instance, multiple projects. Unified "all projects" view with per-project chips.

Source maps

Upload JavaScript source maps to get original file names and line numbers in stack traces.

# Install

Pick the method that fits your setup.

Homebrew
brew tap webwiebe/bugbarn brew install bugbarn
APT (Debian/Ubuntu)
curl -fsSL https://webwiebe.nl/apt/key.gpg \ | sudo gpg --dearmor \ -o /etc/apt/trusted.gpg.d/webwiebe.gpg echo "deb https://webwiebe.nl/apt/ stable main" \ | sudo tee \ /etc/apt/sources.list.d/webwiebe.list sudo apt-get update \ && sudo apt-get install bugbarn
Docker
docker run -d \ -e BUGBARN_API_KEY=your-key \ -e BUGBARN_ADMIN_USERNAME=admin \ -e BUGBARN_ADMIN_PASSWORD=your-password \ -p 8080:8080 \ -v bugbarn-data:/var/lib/bugbarn \ ghcr.io/webwiebe/bugbarn/service:latest

Then open http://localhost:8080 — or read the full getting-started guide for Docker Compose, Kubernetes, and SDK setup.

# How it works

01

Your app sends errors to POST /api/v1/events — accepted immediately, processed async.

02

BugBarn fingerprints, groups, and stores everything in SQLite on your server.

03

Browse issues, set alerts, and get weekly digests — fully in your own infrastructure.

04

Privacy scrubbing runs automatically — emails, IPs, UUIDs, and secrets are redacted before storage.
